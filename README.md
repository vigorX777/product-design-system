# 产品设计系统

> 一套面向产品经理的 AI 辅助产品设计工作流，将 AGENTS.md 作为系统提示词，让 AI 成为你的设计协作伙伴。

---

## 这是什么？

**产品设计系统**是一套可直接 fork 并定制的产品设计工作框架。它的核心思路是：

- 把团队的设计规范、文档标准、交互模式沉淀为结构化的 Markdown 文件；
- 将这些文件作为 AI 助手（Claude、Cursor、ChatGPT 等）的上下文输入；
- 让 AI 在每次协作中自动遵循你的规范，而不是每次从头解释。

你不需要是工程师。只需会用 Markdown 和 AI 工具，就能运转这套系统。

| 角色 | 使用方式 |
|------|----------|
| **产品经理** | 用 AI 辅助完成原型搭建、需求文档编写、设计复盘 |

| **设计规范建设者** | 建立团队级的可复用设计知识库 |

## 核心理念

本系统由四个机制支撑：

### 1. 规范即上下文
`AGENTS.md` 是系统的入口文件，也是 AI 的"工作手册"。它定义了文件结构规则、文档写作标准、工作流程和复盘机制。将它加载到 AI 对话中，AI 就能按照你的规范协作。

### 2. 三件套工作流
每个需求对应一个独立文件夹，包含三个文件：
- **HTML 原型**：单文件交互原型，内联 CSS + JS，无需服务器
- **需求说明文档**：最终态文档，始终反映最新设计
- **变更日志**：精简表格，记录每次变更的时间和摘要

> 需求级变更查看各需求目录下的 `CHANGELOG.md`；系统级变更查看根目录 `系统变更说明.md`。

### 3. 知识库自进化
`product-context/` 是团队的设计知识库。每次需求复盘后，AI 会建议将提炼出的设计原则和交互模式写回知识库，由人工确认后沉淀。随着使用深入，知识库越来越丰富，AI 协作质量也随之提升。

### 4. 两层架构
系统分为**通用层**（跨产品通用的规范和工作流）和**产品专属层**（针对具体产品的业务背景、术语和约束）。fork 后只需填写第二层，通用层开箱即用。

### 5. 模板与本地分离
`template/` 用于存放 GitHub 模板骨架，只保留公开可复用的结构与示例；`product-context/` 用于存放当前本地真实知识库，承载你的实际规范、案例和沉淀内容。日常使用默认读取 `product-context/`，而不是 `template/`。

## 快速开始

### 第一步：Fork 本仓库

```bash
git clone https://github.com/your-username/product-design-system.git
cd product-design-system
```

### 第二步：配置产品专属规则

打开 `AGENTS.md`，找到 `## 二、产品专属规则` 章节，填写：
- 你的产品/平台名称
- 产品线划分
- 使用的前端组件库（如有）
- 目录版本结构

### 第三步：定制知识库

- `product-context/05-业务术语表.md`：填入你的产品领域术语
- `product-context/01-原型设计规范.md`：根据团队技术栈调整原型实现方式
- `product-context/组件模板/`：替换或扩充 HTML 原型模板
- 若你的产品有明确的业务上下文，建议新增类似 `06-产品背景地图.md` 的背景摘要文件，作为 AI 默认优先读取入口

### 第四步：加载到 AI 工具

将 `AGENTS.md` 的内容粘贴或引用到你的 AI 工具上下文中：

- **Cursor**：放入 `.cursor/rules/` 或项目根目录
- **Claude Projects**：作为 Project Instructions 粘贴
- **ChatGPT**：作为自定义指令或对话开头 System Prompt

### 第五步：开始协作

创建第一个需求文件夹，告诉 AI："帮我为「{需求名称}」创建三件套文件"。

## 模板维护说明

- `template/`：用于维护和导出 GitHub 模板文件，不放当前产品的真实规范内容；
- `product-context/`：用于维护当前本地项目实际使用的知识库内容；
- 当你调整系统骨架时，需要同时评估 `template/` 是否需要同步更新；
- 当你补充当前产品经验、术语、背景和案例时，只改 `product-context/`。
- 改完结构后，先运行 `python3 scripts/正式工具/检查系统.py` 再继续迭代。

## 如何定制

| 定制项 | 位置 | 说明 |
|--------|------|------|
| 产品背景 | `AGENTS.md` §二 | 填写平台名、产品线、版本目录结构 |
| 文档写作风格 | `AGENTS.md` §1.2 | 调整界面文案包裹符号、字段描述格式等 |
| 原型视觉风格 | `product-context/01-原型设计规范.md` | 替换色值变量、调整组件样式 |
| 业务术语 | `product-context/05-业务术语表.md` | 填入领域专有词汇，AI 会保持术语一致性 |
| 交互模式 | `product-context/03-交互模式库.md` | 沉淀团队常用的交互流程模式 |
| 设计原则 | `product-context/02-设计原则库.md` | 沉淀从案例中提炼的设计判断原则 |
| 系统级变更 | `系统变更说明.md` | 记录配置文件、结构、模板和正式工具的调整历史 |

## 工作流程一览

```
用户描述需求
     │
     ▼
AI 读取 AGENTS.md + product-context/
     │
     ▼
AI 创建 HTML 原型 + 需求说明文档 + CHANGELOG
     │
     ▼
用户审阅 → 提出修改
     │
     ▼
AI 更新原型 → 更新文档 → 追加日志
     │（多轮迭代）
     ▼
需求完成 → 触发复盘
     │
     ▼
AI 生成复盘文档 → 建议知识库条目
     │
     ▼
用户确认写入 → 知识库积累
```
