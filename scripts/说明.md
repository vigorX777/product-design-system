# 脚本目录说明

`scripts/` 用于集中管理当前系统使用过的脚本，按用途分层。

## 目录结构

```text
scripts/
└── 正式工具/
```

## 目录用途

- `正式工具/`：当前仍在使用、可重复执行的正式脚本。

## 当前正式工具

- `正式工具/整理附件.py`：整理 Obsidian 粘贴截图，按引用文档自动归档；
- `正式工具/转换产品资料.py`：将 `附件/01-产品资料/` 下的 Word、Excel 批量转换为 Markdown。
- `正式工具/检查系统.py`：检查关键目录、关键文件、Office 转 Markdown 完整性、旧路径残留和仓库噪音。

## 使用原则

1. 日常可重复执行的脚本放入 `正式工具/`；
2. 一次性验证脚本和临时改造脚本不长期保留，任务完成后直接删除；
3. 新增脚本前，优先检查现有 `正式工具/` 是否已覆盖该需求，避免重复造轮子。
4. 结构调整后，先补 `系统变更说明.md`，再运行 `python3 scripts/正式工具/检查系统.py`，确认系统仍处于一致状态。
