## 一、关键设计决策回顾

| 决策点 | 最终方案 | 权衡说明 |
|--------|----------|----------|
| 是否展示不可删除项的明细列表 | 只展示数量统计，不展示明细 | 用户只需感知"有多少被跳过"，无需逐条核查，减少弹窗信息噪声 |
| 全不可删时是否保留「删除」按钮 | 仅保留「知道了」，去掉「删除」 | 避免用户误操作触发空删除；阻断式弹窗更清晰地传达"无法执行"语义 |
| 全不可删时是否展示成功 Toast | 不展示任何 Toast | 没有删除发生就没有"成功"可言，保持反馈与实际操作的一致性 |
| 部分可删时的 Toast 文案 | 同时呈现成功数量和跳过数量 | 让用户在操作结束后也能完整了解执行结果，无需重新计算 |
| 规则同时适用两个场景 | 采集任务 + 监控策略共用同一套分支逻辑 | 两者判断依据相同（是否关联监控目标），统一规则降低开发理解成本 |

## 三、可复用的交互模式

### 模式：操作前置分类校验 + 分场景弹窗

**适用场景**：批量操作中存在部分不可执行项时。

**模式描述**：
- 点击批量操作按钮后，先对选中项进行分类统计；
- 根据「全部可执行 / 部分可执行 / 全部不可执行」三种结果，展示不同文案和按钮组合的确认弹窗；
- 全部不可执行时，使用阻断式弹窗（仅「知道了」），不提供操作按钮。

**文案模板**：

| 场景 | 标题 | 副文案 | 按钮 |
|------|------|--------|------|
| 全部可执行 | 「确定批量删除选中的 N 条{对象}吗？」 | 无 | 「删除」「取消」 |
| 部分可执行 | 「确定批量删除选中的{对象}吗？」 | （橙色）选中 N 条，其中 X 条已关联监控目标不可删除，仅删除剩余 Y 条。 | 「删除」「取消」 |
| 全部不可执行 | 「无法删除选中的{对象}」 | 选中的 N 条{对象}均已关联监控目标，无法删除。请先移除关联的监控目标后再试。 | 「知道了」 |

**建议写入**：`product-context/03-交互模式库.md` → 「批量操作前置校验模式」

---

### 模式：操作结果 Toast 区分场景

**适用场景**：批量操作执行后的结果反馈。

**模式描述**：
- 全部成功：「成功删除 N 条{对象}」；
- 部分成功：「成功删除 Y 条{对象}，X 条因{原因}已跳过」；
- 全部未执行：不展示 Toast（阻断式弹窗已传达信息）。

**建议写入**：`product-context/04-文案规范.md` → 「批量操作结果反馈文案」
