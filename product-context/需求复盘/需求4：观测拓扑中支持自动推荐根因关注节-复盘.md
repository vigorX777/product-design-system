# 复盘：需求4：观测拓扑中支持自动推荐根因关注节

**日期**: 2026-03-12
**需求路径**: `v4.5.x/观测中心/中广核项目需求/需求4：观测拓扑中支持自动推荐根因关注节/`
**数据源**: CHANGELOG / 对话历史 / 需求说明文档（_进展备忘.md 不存在）

## 背景

本次需求围绕观测拓扑中的「根因推荐」能力展开，目标是在节点多、告警密集的场景下，为运维人员提供可快速理解、可直接操作的排查入口。设计过程中，该能力逐步与告警回溯形成强关联，最终沉淀为“自动推荐 + 手动重算 + 共用时间上下文”的方案。由于本次未沉淀 `_进展备忘.md`，本复盘主要基于 CHANGELOG、最终需求说明文档和对话决策生成，数据源相对有限。

## 决策清单

| 决策点 | 最终决定 | 决策理由 | 结果评估 |
|--------|----------|----------|----------|
| 推荐触发方式 | 进入页面自动执行一次，同时保留基于当前时间区间的手动「根因推荐」 | 既保留智能辅助的即时价值，又避免完全依赖用户手动触发 | 好 |
| 折叠态状态反馈 | 执行中仅通过按钮光晕反馈，完成后仅显示红点，不再显示状态文案 | 视觉负担更轻，也更贴近拓扑工具区的悬浮控件风格 | 好 |
| 卡片与回溯关系 | 展开根因推荐卡片时同步展开告警回溯面板，时间区间统一由回溯双端手柄控制 | 两个能力本质上共用时间上下文，合并操作路径后理解成本更低 | 好 |
| 分析时间口径 | 根因分析受回溯窗口约束，展示窗口上限 24 小时，告警数据最多向前追溯 15 天 | 兼顾数据可读性、实现复杂度和时间回看场景 | 好 |
| 评分展示策略 | 推荐列表仅显示综合得分徽章，点击后弹出「评分依据」 | 用户更关注“值不值得优先排查”，而不是复杂公式本身 | 好 |
| 评分依据字段收敛 | 仅展示首次告警时间、持续异常时长、告警等级分布、告警总数、所在层级、对象模型类型 | 与开发对齐后确认无需拆分多维权重，保留最小可解释信息即可 | 好 |
| 算法说明位置 | 从卡片内容区挪到标题右侧，分析区间独占整行展示 | 释放时间信息的展示空间，减少头部拥挤感 | 好 |
| 时间展示方式 | 根因推荐卡片中的分析区间统一显示完整日期和时刻，并保持单行呈现 | 避免只显示时分秒带来的歧义，也保证视觉整洁 | 好 |

## 提炼的模式 / 原则（候选）

> 以下为候选条目，经人工确认后追加到对应知识文件。

**设计原则候选**:
- [ ] 智能推荐默认轻打扰: 对智能分析类能力，优先自动执行并用轻量状态反馈结果，而非强制打断用户流程（来源：本次需求）
- [ ] 解释信息按开发可落地粒度收敛: 当算法解释成本高时，优先展示稳定字段和核心依据，不强行暴露复杂评分拆解（来源：本次需求）

**交互模式候选**:
- [ ] 智能推荐与时间回溯共用上下文: 推荐结果面板展开时同步展开时间回溯面板，统一使用一套时间选择入口（来源：本次需求）
- [ ] 折叠态异步任务反馈模式: 悬浮入口在执行中使用动态光晕，完成后使用红点提示有新结果可查看（来源：本次需求）
