# 交互模式库

> 从实际需求中提炼的可复用交互模式，每个模式描述一个具体的用户操作流程。

## 字段说明

| 字段 | 说明 |
|------|------|
| **编号** | IP-001 格式，递增 |
| **名称** | 简洁的模式名称 |
| **触发条件** | 什么情况下使用此模式 |
| **流程描述** | 步骤列表：用户操作 → 系统响应 |
| **状态说明** | 各状态的视觉表现 |
| **异常处理** | 错误/边界情况的处理方式 |
| **适用场景** | 适用的业务上下文 |
| **引用案例** | 引用的需求路径（必填） |

## IP-001: 条件判断→分支弹窗

- **触发条件**: 用户点击列表行的操作按钮（如「配置采集任务」）时。
- **流程描述**:
  1. 用户操作: 点击操作按钮。
  2. 系统响应: 系统后台校验前置条件（如是否已有关联任务）。
  3. 系统响应: 若满足条件A（有），弹出选择弹窗供用户选择已有项；若满足条件B（无），直接跳转至新建页面。
- **状态说明**:
  - 有关联任务: 弹出包含已有任务列表的模态框。
  - 无关联任务: 直接新开窗口进入创建流程。
- **异常处理**: 若校验接口超时或失败，提示"获取关联状态失败，请重试"。
- **适用场景**: 存在多种前置状态，且不同状态对应不同后续操作路径的业务场景。
- **引用案例**: `v4.5.x/监控策略目标显示无采集任务关联信息/`

## IP-002: 关联选择弹窗

- **触发条件**: 用户需要从已有列表中选择一项进行关联或编辑时（如选择已有采集任务）。
- **流程描述**:
  1. 用户操作: 触发选择操作，打开弹窗。
  2. 系统响应: 弹窗内以表格形式展示可选列表，提供关键信息列。
  3. 用户操作: 点击表格行选择目标项。
  4. 系统响应: 关闭弹窗并跳转至目标项的编辑/详情页面。
- **状态说明**:
  - 列表项: 关键名称使用蓝色链接样式，状态使用标签样式。
  - 交互反馈: 鼠标悬浮表格行时背景色变化，点击整行即可触发选择。
- **异常处理**: 列表为空时展示空状态提示，并提供"全新创建"的快捷入口。
- **适用场景**: 需要在当前上下文中快速选择已有资源并跳转处理的场景。
- **引用案例**: `v4.5.x/监控策略目标显示无采集任务关联信息/`

## IP-003: 批量操作前置分类校验

- **触发条件**: 用户勾选多条记录后点击批量操作按钮（如「批量删除」），但所选项中存在因业务约束而不可执行的记录时。
- **流程描述**:
  1. 用户操作: 勾选若干条记录，点击「批量删除」。
  2. 系统响应: 对选中项进行分类统计（可执行数量 / 不可执行数量）。
  3. 系统响应: 根据统计结果分三条分支，展示对应弹窗：
     - **全部可执行**：展示标准确认弹窗，提供「删除」「取消」按钮；
     - **部分可执行**：在弹窗内以橙色警告文字说明跳过数量，提供「删除」「取消」按钮；
     - **全部不可执行**：展示阻断式弹窗，说明原因，仅提供「知道了」按钮。
  4. 用户操作（部分/全部可执行场景）: 点击「删除」确认执行。
  5. 系统响应: 仅对可执行项执行操作，完成后展示结果 Toast。
- **状态说明**:
  - 全部可执行: 标准确认弹窗，无额外提示文案；
  - 部分可执行: 弹窗副文案为橙色（`var(--color-warning)`），明确告知跳过数量和原因；
  - 全部不可执行: 阻断式弹窗，无「删除」按钮，用户只能关闭弹窗，不触发任何删除操作，无结果 Toast。
- **异常处理**: 执行操作后若接口异常，沿用通用错误提示机制；不可执行的判定依据由业务规则决定，不在此模式范围内。
- **适用场景**: 批量操作中存在部分不可执行项，且不可执行原因需明确告知用户的场景（如批量删除已关联资源的记录）。
- **引用案例**: `v4.5.x/批量删除交互优化/`

## IP-004: 侧滑详情查看

- **触发条件**: 用户点击列表行的详情链接或名称时。
- **流程描述**:
  1. 用户操作: 点击列表中的详情入口。
  2. 系统响应: 从页面右侧滑出详情面板（Drawer），覆盖部分当前页面，同时显示半透明遮罩层。
  3. 用户操作: 在侧滑框内查看信息或进行操作。
  4. 用户操作: 点击关闭图标或遮罩层。
  5. 系统响应: 侧滑面板收起，返回原列表视图。
- **状态说明**:
  - 展开状态: 侧滑框占页面三分之一宽度，带有阴影效果，背景有半透明遮罩。
  - 文本截断: 详情列默认文本截断显示，鼠标悬浮弹出 tooltip 展示完整内容。
- **异常处理**: 侧滑框内数据加载失败时，在框内展示错误提示和重试按钮，不影响底层列表。
- **适用场景**: 需要查看详情但不希望用户离开当前列表上下文的场景。
- **引用案例**: `v4.5.x/监控策略目标显示无采集任务关联信息/`

## IP-005: 列表页前置操作引导横条

- **触发条件**: 列表页的某项功能依赖于其他页面的前置配置才能生效，且该前置配置尚未完成或需要用户关注时。
- **流程描述**:
  1. 系统响应: 在列表顶部（搜索栏/操作栏下方、表格上方）渲染一条浅蓝色提示横条。
  2. 横条内容: 左侧 ℹ️ 图标 + 说明文字（说明依赖关系及影响），右侧文字按钮「前往 {目标页面} →」。
  3. 用户操作: 点击文字按钮，跳转至前置配置页面（新标签页或当前页跳转，视业务而定）。
- **状态说明**:
  - 横条背景色：`#E8F4FF`；边框：`1px solid #B3D9FF`；左侧竖条：`3px solid #1272FF`；圆角 `4px`。
  - 跳转按钮使用文字按钮（`.btn-text`）样式，颜色 `#1272FF`，末尾加 `→` 箭头。
- **异常处理**: 若已完成前置配置，横条可根据业务判断是否隐藏（若无法动态判断，可始终展示）。
- **适用场景**: 当前页面功能的有效性依赖另一模块的配置，需主动引导用户完成前置步骤的场景。
- **引用案例**: `v4.5.x/观测中心/中广核项目需求/需求1：应用、服务层支持通过CMDB映射第三方系统的实例【中广核】/`

## IP-006: 多选下拉编辑弹窗

- **触发条件**: 用户点击列表行的「编辑」按钮，需要为该行记录从候选列表中增减多个关联项时。
- **流程描述**:
  1. 用户操作: 点击行操作区「编辑」按钮。
  2. 系统响应: 打开 Modal 弹窗，标题格式为「编辑{功能} — {对象名称}」。
  3. 系统响应: 弹窗内展示多选下拉选择器（MultiSelect），已关联项以 tag 形式显示在输入框内。
  4. 用户操作: 点击输入框展开下拉面板，从选项列表中勾选/取消目标项；或点击 tag 上的「×」移除已选项。
  5. 用户操作: 点击「确定」保存，弹窗关闭并展示 Toast 提示。
- **状态说明**:
  - **锁定项**（系统内置/强制关联）：tag 无「×」删除按钮，下拉选项置灰不可点击（`.is-disabled`）；
  - **可选项**：tag 带「×」可删除，下拉选项可勾选/取消；
  - 两类 tag 使用相同视觉（蓝色 `#E1ECFF` 背景），通过有无 × 按钮区分；
  - 弹窗内 `overflow-y: auto` 时下拉面板需用 `position: fixed` 定位避免被裁切（详见原型规范）。
- **异常处理**: 弹窗内容区较高时，Body 区域独立滚动，Header/Footer 固定；下拉面板始终在视口内可见。
- **适用场景**: 列表行级别的多值关联编辑，被编辑项中存在不可删除的系统锁定项的场景。
- **引用案例**: `v4.5.x/观测中心/中广核项目需求/需求1：应用、服务层支持通过CMDB映射第三方系统的实例【中广核】/`

## IP-007: 异步删除 + 行级状态流转

- **触发条件**: 用户点击列表行的「删除」按钮，且删除操作需要异步处理（如向后端下发停止采集任务再删除）。
- **流程描述**:
  1. 用户操作：点击行操作区「删除」按钮。
  2. 系统响应：在按钮正上方弹出 Popconfirm 气泡确认框（参见 IP-011）。
  3. 用户操作：点击确认按钮。
  4. 系统响应：行进入「删除中」状态——行背景变灰（`#e8e8e8`）、文字变淡（`#999`）、删除按钮文案改为「删除中...」（禁用）、编辑按钮禁用、复选框禁用、开关自动关闭。
  5. 系统响应（成功）：行淡出并向左移动消失（`opacity: 0, transform: translateX(-20px)`，0.3s ease），DOM 移除，列表总条数 -1。
  6. 系统响应（失败）：行样式完全恢复（含开关恢复至操作前状态），实例显示名列前出现红色感叹号图标（14×14px，`#ff4d4f`），鼠标悬浮 Tooltip 显示「停止采集任务失败，请重试」，删除按钮恢复可点击。
- **状态说明**:
  - 删除中：行背景 `#e8e8e8`，文字 `#999`，按钮文案「删除中...」且 `disabled`；
  - 删除成功：行淡出左移动画后 DOM 移除；
  - 删除失败：行完全恢复原样，行内行级失败标识（红色感叹号 + Tooltip）。
- **异常处理**: Popconfirm 支持点击取消按钮、外部区域点击、ESC 键三种关闭方式，均不触发删除。
- **适用场景**: 需要异步处理的列表行删除，尤其是删除前需先停止/解绑关联资源的场景。
- **引用案例**: `v4.5.x/监控中心/采集任务删除效果优化/`

## IP-008: 操作锁定 + 顶部 Banner + 自动恢复轮询

- **触发条件**: 用户触发异步操作（启停开关、删除等），操作需等待后端处理完成后才能继续。
- **流程描述**:
  1. 用户操作：触发操作（如切换启停开关）。
  2. 系统响应：局部锁定数据操作入口——新增按钮、批量操作按钮、复选框、操作列按钮均禁用；侧边栏、搜索框、分页保持可用。
  3. 系统响应：当前行状态列替换为 loading 图标 + 「任务下发中...」文案。
  4. 系统响应：顶部 Banner 展开——橙色背景（`#FFF3E0`），带呼吸动效，标题「任务下发中」，副标题说明锁定原因，右侧提供「手动刷新」快捷链接。
  5. 系统响应：5 秒后自动轮询刷新一次，后端完成则 Banner 消失、所有入口解锁、行状态恢复正常显示。
- **状态说明**:
  - 锁定期间：数据操作按钮全部 `disabled`，当前行显示 loading；
  - Banner 状态：橙色警示色 + 呼吸动效（`keyframes: opacity 1.5s ease-in-out infinite`），包含「手动刷新」链接；
  - 解锁后：Banner 淡出消失，所有入口恢复正常。
- **异常处理**: 轮询超时或网络异常时，Banner 保留，用户可点击「手动刷新」主动触发状态更新，不自动重复轮询。
- **适用场景**: 后端处理需要数秒到数十秒的异步操作，需防止用户在等待期间重复操作导致数据不一致。
- **引用案例**: `v4.5.x/监控中心/采集任务操作逻辑优化/`

## IP-009: 增量数据合并选中

- **触发条件**: 用户从来源页面（如异常监控目标列表）批量选择数据，带入目标页面（如采集目标列表）中。
- **流程描述**:
  1. 用户操作：在来源页面选择若干条数据，点击确认带入。
  2. 系统响应：自动与目标列表中的已有数据去重合并——重复项不重复插入。
  3. 系统响应：新增行置顶显示，浅蓝色背景高亮（区别于已有行），并自动勾选（`checkbox` 选中）。
  4. 系统响应：展示 Toast 提示「已从{来源}中新增 N 个{对象}，并已自动选中」。
- **状态说明**:
  - 新增行高亮：浅蓝色背景（`#E8F4FF`），置顶排列；
  - 重复行：不高亮、不置顶，保持原有位置；
  - Toast 类型：成功（绿色），展示新增数量。
- **异常处理**: 若带入的数据全部与已有数据重复，不展示高亮行，Toast 提示「所选数据已在列表中，无需重复添加」。
- **适用场景**: 跨页面批量选择并并入当前列表的场景，尤其是需要让用户立即确认哪些是新增的内容。
- **引用案例**: `v4.5.x/监控中心/监控策略目标显示无采集任务关联信息/`

## IP-010: 树形表格分组管理

- **触发条件**: 列表数据量大，需要按文件夹/分组进行层级归类管理时。
- **流程描述**:
  1. 系统响应：将平铺列表改造为两级树形结构——一级为文件夹行（分组），二级为子项行（如仪表盘）。
  2. 系统响应：默认折叠所有分组，仅展示文件夹行。
  3. 用户操作：点击文件夹行展开图标（▶，展开后旋转 90°），展开该分组下的子项行。
  4. 用户操作（新增分组）：点击「新增分组」按钮 → Modal 弹窗（仅「分组名称」单一输入框，不能为空）→ 确认后创建空文件夹行。
  5. 用户操作（批量移组）：多选子项行 → 点击「批量分组」→ 弹窗选择目标分组 → 确认后子项移入目标分组。
- **状态说明**:
  - 文件夹行：不同背景色（区别于子项行）+ 加粗字体 + 左侧 ▶ 展开图标；
  - 子项行：名称列左侧缩进体现层级关系；
  - 展开状态：▶ 旋转 90°（变为 ▼）。
- **异常处理**: 分组名称为空时，确认按钮禁用；名称重复时，输入框下方显示错误提示「该分组名称已存在」。
- **适用场景**: 仪表盘列表、采集插件库、监控模板等资源量较多且需要分类管理的列表页。
- **引用案例**: `v5.1/监控中心/仪表盘新增分组/`

## IP-011: Popconfirm 气泡轻量确认

- **触发条件**: 单条记录的危险操作（如删除）需要二次确认，但希望避免全屏 Modal 打断用户上下文。
- **流程描述**:
  1. 用户操作：点击操作按钮（如「删除」）。
  2. 系统响应：在按钮正上方弹出气泡确认框，底部带箭头指向触发按钮。
  3. 用户操作：点击红色确认按钮 → 执行操作；或点击「取消」/外部区域/ESC → 气泡关闭，无任何操作。
- **状态说明**:
  - 气泡规格：固定宽度 320px，白色背景 + 阴影，距触发按钮 12px；
  - 气泡内容：左上角黄色警告图标（`#faad14`）+ 说明文字，底部右对齐「取消」（文字按钮）+「确认」（红色危险按钮，`#ff4d4f`）；
  - 关闭方式：点击确认、点击取消、点击气泡外区域、按 ESC，四种均可关闭。
- **异常处理**: 若确认按钮触发的操作失败，气泡已关闭，失败反馈由行级状态（参见 IP-007）或 Toast 承接。
- **适用场景**: 相较于 Modal 弹窗，适合单行操作的轻量二次确认，减少视觉打断；不适合需要填写额外信息的确认场景。
- **引用案例**: `v4.5.x/监控中心/采集任务删除效果优化/`

## IP-012: 集群菱形双态节点

- **触发条件**: 拓扑图中某层（如服务层、中间件层、数据库层）的节点以「集群」为单位聚合展示时。
- **流程描述**:
  1. 系统响应（折叠态）：集群节点以菱形样式展示，节点右上角显示聚合总数 badge（如「×9」），连线指向菱形节点。
  2. 用户操作：双击菱形节点，触发展开。
  3. 系统响应（展开态）：原菱形节点位置拆分为两行——上半行显示集群菱形小节点（代表集群本身，尺寸缩小），下半行显示该集群下的普通实例节点；连线动态重绘，指向各实例节点。
  4. 用户操作：再次双击集群菱形小节点，触发折叠，恢复为折叠态。
  5. 用户操作（悬浮）：鼠标悬浮在折叠态菱形节点上，显示「聚合卡片」（仅展示规模/告警总数，不含实例明细）；悬浮在展开态实例节点上，显示「实例卡片」（含实例字段详情）。
- **状态说明**:
  - 折叠态：菱形节点，右上角 badge 显示「×N」聚合数量；
  - 展开态：上半行小菱形（集群本身）+ 下半行普通圆形/方形实例节点；
  - 悬浮卡片：折叠态聚合卡片不显示实例字段，仅展开态实例卡片显示完整字段——信息层次随用户操作意图递进。
- **异常处理**: 展开后实例数量超出当前行宽时，自动换行排列；连线数量过多时，使用弯曲路径避免遮挡菱形节点。
- **适用场景**: 拓扑图中存在集群聚合关系（如 K8s 集群 → Pod 实例），需要在「总览」和「诊断」两种视图之间切换的场景。

## IP-013: 嵌入式侧滑抽屉集成

- **触发条件**: 综合原型中需要集成独立功能模块（如配置抽屉、帮助面板）时，且该模块的 CSS/JS 与主页存在命名冲突风险。
- **流程描述**:
  1. 用户操作：点击工具栏图标（如「⚙ 集群聚合规则」按钮）。
  2. 系统响应：从页面右侧滑出抽屉面板（`transform: translateX(0)`），背景出现半透明遮罩。
  3. 系统响应：抽屉内所有 CSS 类名使用统一前缀（如 `.agg-*`）与主页完全隔离，不会影响主页样式。
  4. 用户操作：在抽屉内完成配置操作后，点击「保存」或关闭图标。
  5. 系统响应：抽屉收起（`transform: translateX(100%)`），遮罩消失，主页状态保持不变。
- **状态说明**:
  - 收起态：抽屉在视口外，`transform: translateX(100%)`；
  - 展开态：抽屉覆盖页面右侧，宽度通常为 480px；
  - CSS 命名空间：抽屉内所有样式类使用前缀隔离（如 `.agg-drawer`、`.agg-form`、`.agg-btn`），避免与主页的 `.drawer`、`.form`、`.btn` 等通用类名冲突。
- **异常处理**: 抽屉内表单未保存时，若用户点击遮罩或关闭按钮，弹出二次确认「是否放弃未保存的修改？」。
- **适用场景**: 将独立开发的功能模块（如规则配置抽屉）集成进综合原型，且两者样式不能相互污染的场景。
- **引用案例**: `v4.5.x/观测中心/中广核项目需求/需求1：应用、服务层支持通过CMDB映射第三方系统的实例【中广核】/`
