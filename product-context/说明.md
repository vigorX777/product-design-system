# 产品设计知识库

> 本目录是当前本地项目实际使用的知识积累中心，沉淀监控中心、观测中心及具体需求中验证过的规范、模式、术语和复盘内容。

> `template/product-context/` 存放 GitHub 模板骨架；当前目录 `product-context/` 存放本地真实内容，日常协作默认读取当前目录。

## 目录结构

| 文件/目录 | 用途 | 更新频率 |
|-----------|------|----------|
| 01-原型设计规范.md | 原型技术路线与视觉规范 | 按需（技术决策变更时） |
| 02-设计原则库.md | 从实际案例提炼的可复用设计原则 | 每次需求复盘后 |
| 03-交互模式库.md | 从实际需求提炼的交互流程模式 | 每次需求复盘后 |
| 04-文案规范.md | 界面文案与提示语规范 | 按需 |
| 05-业务术语表.md | 产品领域专业词汇与概念定义 | 按需 |
| 06-监控中心背景地图.md | 监控中心设计上下文摘要 | 按产品资料更新 |
| 07-观测中心背景地图.md | 观测中心设计上下文摘要 | 按产品资料更新 |
| 组件模板/ | HTML 原型模板 | 按需 |
| 需求复盘/ | 复盘日记 | 每个需求完成后 |

## 维护边界

- 改系统骨架、模板占位和公开示例：到 `template/product-context/` 维护；
- 改当前产品真实规范、术语、案例、背景知识：在当前目录维护；
- 如当前目录新增了通用能力，需人工判断是否值得回收到模板骨架，而不是直接覆盖模板。

## 知识治理规则

1. **准入条件**: 每条知识条目必须引用至少 1 个实际案例；能提升下一次生成质量
2. **更新方式**: 半自动——系统建议候选条目，人工确认后写入
3. **冲突处理**: 产品专属规则 > 通用规则 > 默认值
4. **知识生命周期**: 条目可标记为"已废弃"，不删除历史记录

## 快速入口

- 做监控中心需求 → 先读 `06-监控中心背景地图.md` → 再读 `05-业务术语表.md` → 再补 `02-设计原则库.md` / `03-交互模式库.md`
- 做观测中心需求 → 先读 `07-观测中心背景地图.md` → 再读 `05-业务术语表.md` → 再补 `02-设计原则库.md` / `03-交互模式库.md`
- 新建需求原型 → 参考 `01-原型设计规范.md` + `组件模板/`
- 复盘提炼 → 使用 `需求复盘/模板.md`
