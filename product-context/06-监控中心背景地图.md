# 监控中心背景地图

> 本文件用于为需求设计提供监控中心的稳定背景上下文。涉及监控中心相关需求时，优先阅读本文件，再结合术语表、设计原则库和交互模式库补充细节。

## 一、产品定位

嘉为蓝鲸监控中心是嘉为蓝鲸可观测方案中的统一监控工具，面向应用、主机、数据库、中间件、云资源、网络服务、硬件设备等对象提供统一采集、统一检测、统一告警和统一展示能力。它承担的是“广覆盖监控底座”角色，重点是把不同类型资源拉入统一监控体系，而不是强调分层拓扑和根因推理。

## 二、核心页面地图

1. 首页：
   - 展示活动告警、告警趋势、MTTA/MTTR、资源告警占比、监控覆盖率等总览信息。
2. 视图：
   - 「仪表盘」用于统一查看监控、日志、告警数据源与图表配置；
   - 「网络拓扑」用于手工维护和展示网络拓扑、链路与告警状态。
3. 场景：
   - 包括业务资源概览、网站服务拨测、主机监控、云平台监控、K8s 监控、硬件监控、自定义监控场景；
   - 场景页支持下钻到实例详情，查看指标、告警、配置和实例信息。
4. 告警：
   - 包括活动告警、历史告警、告警详情、告警分析与告警处理。
5. 集成：
   - 包括监控插件、监控源插件等第三方或扩展能力接入入口。
6. 管理：
   - 包括接入指引、对象模型、动态分组、监控策略、采集节点、自定义接入、参数集管理。
7. 设置：
   - 包括系统设置、用户和组、操作日志、监控设置。

## 三、关键能力边界

1. 对象模型管理：
   - 支持对接 CMDB 模型创建监控对象模型，并同步纳管实例；
   - 也支持新建对象模型，但只有关联 CMDB 的对象模型才具备实例体系。
2. 动态分组：
   - 仅支持基于已关联 CMDB 的对象模型创建；
   - 用于授权、采集任务和监控策略的动态应用。
3. 数据采集：
   - 支持脚本、SQL、Exporter、DataDog、JMX、BK-Pull、SNMP 等插件；
   - 第三方监控源接入当前内置重点是 Kafka 监控源，其他来源通常需要额外适配。
4. 检测与告警：
   - 支持丰富的检测条件、异常算法、无数据检测和告警等级管理；
   - 当前不支持 AI 类智能检测算法。
5. 网站服务拨测：
   - 支持 HTTP(S)、ICMP、TCP、UDP；
   - 不支持多步骤拨测，不支持页面点击模拟。
6. 云平台与容器：
   - 云平台监控强调统一接入和资源关联；
   - 容器监控以 K8s 平台为前提，并支持容器 node 与主机 host 关联。
7. 可视化：
   - 仪表盘适合多数据源统一展示；
   - 网络拓扑更偏网络资源和链路可视化，准确率受 SNMP 拓扑发现能力限制。

## 四、关键对象与术语

1. 对象模型：
   - 监控中心中定义监控对象的基础单元，决定资源类型、指标和实例组织方式。
2. 动态分组：
   - 基于 CMDB 属性条件动态圈定的一组监控对象实例。
3. 采集任务：
   - 负责定义采集插件、采集参数和采集目标，是采集配置的执行载体。
4. 监控策略：
   - 负责目标范围、检测规则、告警规则和告警处理配置。
5. 采集节点：
   - 远程采集主机，承担拨测、云监控、监控源接入、硬件监控等任务执行能力。
6. 监控源插件：
   - 用于接入第三方监控数据的插件机制。
7. 权限空间：
   - 用于对策略、插件、动态分组等资源做隔离与授权的管理维度。

## 五、已知设计约束

1. 很多配置能力以前置关系为主：
   - 对象模型、动态分组、采集任务、监控策略之间存在明显依赖链；
   - 设计相关页面时，应优先考虑前置操作引导，而不是只展示“不可用”状态。
2. CMDB 是多个核心能力的基础：
   - 对象模型实例、动态分组、业务资源概览等能力均依赖 CMDB 打通；
   - 需求设计中如果出现资源选择、实例展示、动态范围等场景，应优先确认 CMDB 口径。
3. 采集与检测是分层配置：
   - 采集任务负责“数据怎么采”，监控策略负责“数据怎么判”；
   - 需求设计时不宜混淆两类配置职责。
4. 监控中心更偏“统一监控平台”：
   - 强项是覆盖面、接入能力、统一管理；
   - 复杂根因分析、全栈链路诊断、分层拓扑联动更偏观测中心。
