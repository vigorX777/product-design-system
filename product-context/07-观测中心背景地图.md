# 观测中心背景地图

> 本文件用于为需求设计提供观测中心的稳定背景上下文。涉及观测中心相关需求时，优先阅读本文件，再结合术语表、设计原则库和交互模式库补充细节。

## 一、产品定位

嘉为蓝鲸观测中心是嘉为蓝鲸可观测方案中的核心观测能力，强调从业务视角统一观察应用、服务、组件、容器和基础设施的运行状态。它的重点不只是“有没有监控”，而是通过全景观测、分层拓扑、资源详情和告警回溯，帮助用户理解依赖关系、影响范围和故障传播路径。

## 二、核心页面地图

1. 业务全景观测：
   - 提供全业务面板、业务监控覆盖率、告警汇总视图；
   - 支持从业务卡片下钻到业务详情。
2. 业务纵览：
   - 展示资源分类看板和对象模型看板；
   - 支持按业务、资源类型、对象模型查看资源和告警概况。
3. 观测拓扑：
   - 展示业务下监控对象实例的全局拓扑；
   - 支持监控视图、告警视图、节点聚合、搜索、高亮上下游路径与节点下钻。
4. 资源详情：
   - 展示拓扑信息、告警常驻、基础监控、日志监控、监控配置、实例详情。
5. 告警拓扑分析：
   - 在告警详情和关联分析中，从告警视角查看业务的拓扑与影响范围。

## 三、关键能力边界

1. 业务全景与纵览：
   - 强调以业务为入口聚合资源、告警、覆盖率和对象模型信息；
   - 对象模型看板和资源分类看板都依赖对象模型与 CMDB 的关联基础。
2. 观测拓扑：
   - 展示八大类分层架构，包括 APM 应用、组件、基础资源、硬件资源等；
   - 支持内置/自定义聚合、节点搜索、路径高亮、监控/告警染色。
3. 聚合规则：
   - 当前同一业务下、同一对象模型仅支持配置一条 CMDB 聚合规则；
   - 当前不支持基于 CMDB 业务拓扑直接配置聚合规则。
4. 资源详情与告警回溯：
   - 支持查看上下游拓扑、对象与关联资源告警、日志、监控配置、实例信息；
   - 告警回溯能力存在，但当前版本在进度展示和感知强度上仍有待增强。
5. APM/CMDB 关联：
   - 当前 APM 关联组件主要面向内置对象模型；
   - 后续才计划支持关联自定义对象模型。

## 四、关键对象与术语

1. 观测拓扑：
   - 用于展示业务下所有监控对象实例架构关系的分层拓扑视图。
2. 对象模型：
   - 定义被观测对象的模型基础，是资源实例、对象模型看板和聚合规则的重要前提。
3. IP 映射字段：
   - 用于将 APM 探针发现的组件实例与 CMDB 组件实例建立匹配关系。
4. 组件关联：
   - 用于建立 APM 组件与 CMDB 组件的对应关系。
5. 外部组件：
   - 当关联的 CMDB 实例不属于当前业务时，拓扑中保留展示并标记为外部。
6. 告警回溯：
   - 按告警流转过程动态染色拓扑节点，回看历史告警发展过程的能力。
7. 链路（Trace）：
   - 一次分布式调用的完整链路轨迹，是应用性能观测的重要基础对象。

## 五、已知设计约束

1. 设计时要优先考虑“业务视角”：
   - 观测中心不是单资源监控页集合，而是围绕业务、对象模型和拓扑关系组织能力；
   - 页面设计应优先保证业务入口、业务上下文和资源联动的一致性。
2. 拓扑是核心心智模型：
   - 节点、层级、聚合、路径高亮和下钻是核心体验，不应被过多配置噪音干扰；
   - 新需求涉及节点状态、聚合逻辑、回溯时间等时，需要优先保证拓扑认知连续性。
3. 多处能力依赖前置打通：
   - CMDB、APM、日志、容器平台等接入状态直接影响观测能力是否可用；
   - 需求设计中应明确哪些页面是“观测结果页”，哪些能力依赖前置配置。
4. 观测中心更偏“诊断与关联分析”：
   - 相比监控中心强调统一接入与统一策略，观测中心更强调资源关系、影响范围和诊断路径；
   - 需求设计中若涉及“为什么异常”“影响到谁”“上下游关系”，优先放在观测中心语境下思考。
