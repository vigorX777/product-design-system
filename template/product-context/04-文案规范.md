# 文案规范

> 界面文案的统一规则，确保产品表达一致、清晰。模板中的规则为通用起点，可根据你的团队语言风格继续裁剪。

## 格式规则

- 界面文案（按钮名、标题、提示语）统一用「」书名号包裹（在需求文档中引用时）
- 用 `>` 引用块展示 UI 上实际显示的文案原文
- 文案中不出现技术术语（HTTP 状态码、接口名等）

## 按钮命名规范

| 操作类型 | 标准文案 | 说明 |
|----------|----------|------|
| 新建 | 「新建」 | 创建新的业务对象 |
| 编辑 | 「编辑」 | 修改已有的业务对象 |
| 删除 | 「删除」 | 移除业务对象（危险操作） |
| 保存 | 「保存」 | 提交并保存当前表单的修改 |
| 取消 | 「取消」 | 放弃当前操作并关闭弹窗或返回上一页 |
| 确认 | 「确认」 | 在二次确认对话框中同意执行操作 |
| 详情 | 「查看详情」 | 查看业务对象的详细信息 |
| 添加 | 「添加」/「+ 添加」 | 在列表或表单中增加一项 |
| 批量删除 | 「批量删除」 | 同时删除多个选中的业务对象 |
| 重试 | 「重试」 | 重新执行失败的任务或操作 |

## 提示文案规范

| 场景 | 文案模板 | 示例 |
|------|----------|------|
| 操作成功 | `{操作对象}已{动作}` 或 `{动作}成功` | "保存成功"、"删除成功" |
| 引导操作 | `{当前状态说明}，请{操作指引}` | "当前记录尚未配置，请先完成配置后再执行此操作。" |
| 状态说明 | `{当前状态描述}，请{操作指引}` | "存在多个可选项，请选择要编辑的条目，或新建一个。" |
| 依赖提示 | `{前置条件}完成后，{后续操作}才可{动作}` | "基础配置完成后，高级选项才可填写。" |
| 批量操作结果 | `已{动作} {数量} 条{对象}` | "已删除 3 条记录。" |

## 错误信息规范

| 场景 | 文案模板 | 示例 |
|------|----------|------|
| 必填未填 | `请输入{字段名}` 或 `请选择{字段名}` | "请输入名称"、"请选择类型" |
| 格式错误 | `{字段名}格式不正确，请{正确格式说明}` | "邮箱格式不正确，请输入有效的邮箱地址" |
| 唯一性冲突 | `该{字段名}已存在，请重新输入` | "该名称已存在，请重新输入" |
| 关联限制 | `当前{对象}已被{关联对象}引用，无法{动作}` | "当前记录已被其他配置引用，无法删除" |

**原则**：
- 用"请..."引导用户操作，不用"请您..."（过于客气）
- 说明原因时用"...，请...（原因 + 操作指引）"
- 不出现 error、exception、null 等技术词汇

## 空状态文案

| 场景 | 主文案 | 副文案（可选）|
|------|--------|--------------|
| 列表无数据 | 暂无数据 | — |
| 搜索无结果 | 未找到相关内容 | 请尝试其他关键词 |
| 暂无权限 | 暂无查看权限 | 请联系管理员申请权限 |

## 确认对话框文案

| 场景 | 标题 | 内容文案 |
|------|------|----------|
| 删除确认 | 确认删除 | 删除后将无法恢复，确认删除「{对象名}」？ |
| 批量删除 | 确认批量删除 | 已选择 N 项，删除后将无法恢复，是否继续？ |
| 离开未保存 | 确认离开 | 当前页面有未保存的修改，离开后将丢失，是否确认离开？ |

## 批量操作分场景文案

批量操作存在部分不可执行项时，弹窗标题、副文案和结果提示均需分场景表达，不可统一使用"操作成功"。

| 场景 | 标题 | 副文案 | 按钮 |
|------|------|--------|------|
| 全部可执行 | 「确定批量删除选中的 N 条{对象}吗？」 | 无 | 「删除」「取消」 |
| 部分可执行 | 「确定批量删除选中的{对象}吗？」 | （警告色）选中 N 条，其中 X 条因{原因}无法执行，仅删除剩余 Y 条。 | 「删除」「取消」 |
| 全部不可执行 | 「无法删除选中的{对象}」 | 选中的 N 条{对象}均因{原因}无法删除，请先解决后再试。 | 「知道了」 |

**原则**：
- 部分成功场景必须同时说明成功数量和跳过数量；
- 全部不可执行场景禁止展示"成功"类提示；
- 阻断式弹窗中必须说明原因并给出解决方向。

## 操作进行中文案

| 状态 | 文案格式 | 示例 |
|------|---------|------|
| 操作进行中 | 「{操作动词}中...」 | 「删除中...」「提交中...」「保存中...」 |

**原则**：直接在原按钮文案后加「中...」；按钮同时禁用，防止重复点击。
