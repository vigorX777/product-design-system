# 设计原则库

> 从实际需求案例中提炼的可复用设计原则。模板中仅保留字段结构和填写方式，不包含真实项目案例。

## 字段说明

| 字段 | 说明 |
|------|------|
| **编号** | DP-001 格式，递增 |
| **名称** | 简洁的原则名称（4-10 字） |
| **描述** | 1-3 句话说明原则内容 |
| **来源案例** | 引用的需求路径（必填） |
| **适用范围** | 适用的场景或组件类型 |
| **反例** | 违反此原则的典型做法 |

---

> **模板说明**：请在引入到你的团队或产品后，将真实复盘中确认可复用的原则追加到本文件中。不要保留示例占位内容作为正式规范。

## 示例条目（可替换）

## DP-001：示例原则名称

- **描述**：在此填写一条从真实需求中提炼出的设计原则描述，控制在 1-3 句话内。
- **来源案例**：`vX.Y/产品线/需求名称/`
- **适用范围**：在此填写适用场景，如列表页、弹窗、多步骤流程、分析面板等。
- **反例**：在此填写一个违反该原则的典型做法，帮助团队理解边界。
