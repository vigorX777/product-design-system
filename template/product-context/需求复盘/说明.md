# 复盘日记

> 每个需求完成后的复盘记录，用于提炼经验、持续改进设计系统。

## 触发时机

每个需求的原型和文档完成后，进行一次复盘。

## 复盘流程

1. **回顾决策**: 本次需求做了哪些关键设计决策？
2. **提炼模式**: 有哪些交互模式或设计原则可以复用？
3. **更新知识库**: 确认后追加到对应知识文件（人工确认）
4. **记录改进点**: 下次可以做得更好的地方

## 文件命名

格式: `{需求名称}-复盘.md`
示例: `批量删除交互优化-复盘.md`

## 关于"经验提炼"

- 系统在复盘时会建议候选知识条目
- 由产品经理确认后才写入 `02-设计原则库.md` 或 `03-交互模式库.md`
- 确认写入的条目需标注来源案例
