# 原型设计规范

> 本文档定义 HTML 交互原型的技术路线和视觉规范。模板中提供的是通用骨架，请根据你的产品组件库和前端约束替换为真实规范。

## 技术路线

**推荐方案**：纯 HTML 单文件，内联 CSS + JS，无外部依赖。

**核心原则**：
- 原型文件自闭合——可直接在浏览器中双击打开，无需本地服务器；
- 无外部网络依赖——不依赖 CDN，离线可用；
- 视觉风格贴近真实产品——使用 CSS 变量模拟产品组件库的设计 Token；
- 单文件交付——发给开发/测试/评审方时只需传一个文件。

> **定制说明**：如果你的产品组件库（如 Element Plus、Ant Design Vue）提供了可通过 CDN 直接引入的独立构建版本，也可在原型中直接引用，无需坚持纯 CSS 手绘路线。核心目标是让原型可以直接在浏览器打开，而不是限定具体实现方式。

## CSS 变量规范

使用 CSS 变量定义产品的设计 Token，确保所有原型视觉风格一致。以下为通用模板，请根据你的产品组件库实际色值替换：

```css
:root {
  /* 品牌色 —— 替换为你的产品主色调 */
  --color-primary:   #1272FF;  /* 主色 */
  --color-success:   #27C274;  /* 成功色 */
  --color-warning:   #FF9214;  /* 警告色 */
  --color-danger:    #F43B2C;  /* 危险色 */

  /* 文字色 */
  --color-text-primary:    rgba(0, 0, 0, 0.85);
  --color-text-secondary:  rgba(0, 0, 0, 0.45);
  --color-text-disabled:   rgba(0, 0, 0, 0.25);

  /* 边框 & 背景 */
  --color-border:      #D1D7E1;
  --color-bg-base:     #F5F7FA;
  --color-bg-white:    #FFFFFF;

  /* 尺寸 */
  --border-radius:     4px;
  --btn-height-normal: 32px;
  --font-size-base:    14px;

  /* 字体栈 */
  --font-family: -apple-system, BlinkMacSystemFont, 'PingFang SC', 'Microsoft YaHei', sans-serif;
}
```

## 文件结构模板

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>{原型标题}</title>
  <style>
    /* CSS 变量定义 */
    /* 全局重置与基础样式 */
    /* 布局样式 */
    /* 组件样式（按钮、表格、弹窗等） */
    /* 页面特定样式 */
  </style>
</head>
<body>
  <!-- 原型内容 -->
  <script>
    /* 交互逻辑（事件监听、状态管理、动态渲染等） */
  </script>
</body>
</html>
```

## 组件实现说明

原型中常用的 UI 组件可通过纯 CSS 实现，包括但不限于：

- **按钮**：主按钮、次按钮、危险按钮、文字按钮、图标按钮
- **表格**：列表、排序、分页、行选择（单选/多选）、操作列
- **弹窗**：Modal 确认框、侧滑 Drawer、气泡确认 Popconfirm
- **表单**：输入框、下拉选择、多选框、日期选择
- **反馈**：Toast 提示、Loading 状态、空状态、错误状态
- **导航**：侧边栏、顶部导航、面包屑、标签页

> 具体 CSS 实现代码请参考 `组件模板/` 目录下的模板文件，可直接复制使用。

## 限制

- 不使用 webpack / vite / npm 等构建工具；
- 单文件 HTML，内联 CSS + JS；
- 不在原型文件中描述实现细节（如具体 CSS 类名、函数名），原型只体现产品行为。
