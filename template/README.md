# 产品设计系统模板

> 一套面向产品经理的 AI 辅助产品设计工作流模板。模板只提供结构骨架、字段定义和最小示例，不包含任何真实业务规范内容。

---

## 这是什么？

这是一个可直接 fork 的产品设计系统模板，用来帮助你搭建自己的 AI 协作式产品设计工作区。它的目标不是提供现成的业务答案，而是提供一套可持续迭代的结构：

- 用 `AGENTS.md` 约束 AI 的协作方式；
- 用 `product-context/` 沉淀规范、术语、模式和模板；
- 用需求目录管理原型、需求说明和变更日志；
- 用复盘机制把经验持续回流到知识库。

如果你希望把产品设计过程从“靠记忆和聊天记录推进”变成“靠结构化文件推进”，这个模板就是起点。

---

## 模板提供什么？

本模板默认提供以下骨架：

- `AGENTS.md`
  - 产品设计协作规则模板
  - 包含需求文件结构、文档规范、复盘机制、附件管理规则等基础约定
- `product-context/`
  - `01-原型设计规范.md`
  - `02-设计原则库.md`
  - `03-交互模式库.md`
  - `04-文案规范.md`
  - `05-业务术语表.md`
  - `组件模板/`
  - `需求复盘/模板.md`
- `README.md`
  - 模板使用说明

这些文件只定义“应该有哪些内容、每类内容如何组织”，不会替你填入真实业务规范。

---

## 模板不包含什么？

模板明确不包含以下内容：

- 真实产品术语
- 客户资料或项目资料
- 内部设计规范成品
- 实际业务背景地图
- 私有原型、私有截图、私有用户手册
- 已验证的真实设计原则和交互模式案例

也就是说，模板是“骨架”，不是“你的成品仓库”。

---

## 如何使用？

建议按以下顺序落地：

### 1. Fork 仓库

```bash
git clone https://github.com/your-username/product-design-system.git
cd product-design-system
```

### 2. 填写 `AGENTS.md`

优先补齐以下内容：

- 产品/平台名称
- 产品线划分
- 前端技术栈或设计系统约束
- 版本目录结构

### 3. 补充知识库基础内容

至少补充以下文件：

- `product-context/05-业务术语表.md`
- `product-context/01-原型设计规范.md`
- `product-context/04-文案规范.md`

### 4. 调整组件模板

根据你的页面风格和组件约束，调整：

- `product-context/组件模板/`

### 5. 按需新增产品背景地图

如果你的产品存在明显的业务背景和页面体系，建议新增类似：

- `product-context/06-产品背景地图.md`

让 AI 在处理需求时先读背景，再读原则和模式。

### 6. 开始按需求目录协作

每个需求创建独立文件夹，维护：

- 原型 HTML
- 需求说明文档
- `CHANGELOG.md`

## 如何定制？

| 定制项 | 位置 | 说明 |
|--------|------|------|
| 产品背景 | `AGENTS.md` §二 | 填写平台名、产品线、版本目录结构 |
| 文档写作风格 | `AGENTS.md` §一 | 调整文案包裹、字段描述、章节写法等 |
| 原型视觉风格 | `product-context/01-原型设计规范.md` | 替换色值、组件风格、实现边界 |
| 业务术语 | `product-context/05-业务术语表.md` | 填入领域术语，保持 AI 输出一致 |
| 交互模式 | `product-context/03-交互模式库.md` | 沉淀真实项目中复用的交互流程 |
| 设计原则 | `product-context/02-设计原则库.md` | 沉淀从需求复盘中提炼的判断原则 |
| 附件管理 | 可按需新增 `附件/README.md` | 约束用户手册、交底书、截图和素材的归档方式 |

---

## 模板边界

- `template/` 或模板仓库中的内容只负责提供结构骨架；
- 真实项目规范应在你自己的实例仓库中维护；
- 不要把客户资料、真实产品规则、私有术语直接写回模板仓库；
- 当结构骨架变化时，优先更新模板，再同步更新你的实例仓库。

---

## 建议补充的实例能力

在你的实际项目仓库中，建议继续补这几类内容：

- 产品背景地图
- 附件管理规则
- 系统变更说明
- 正式工具脚本说明

这样模板落地后，才会真正变成可持续运转的系统，而不是一套静态文件。
